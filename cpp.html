        • What Is a Variable?

          In C++, a variable is a named storage location that holds a value. Variables are used to store and manipulate data within a program. To define a variable in C++, you need to specify its data type and a name. The basic syntax for defining a variable is as follows:

          data_type variable_name;

          Here's an example:

          #include <iostream>

          int main()
          {
              // Define an integer variable named 'age'
              int age;

              // Assign a value to the 'age' variable
              age = 25;

              // Display the value of the 'age' variable
              std::cout << "Age: " << age << std::endl;

              return 0;
          }

          You can also combine variable declaration and assignment in a single statement:

          #include <iostream>

          int main()
          {
              int age = 25;
              return 0;
          }

          Listing 3.1 Using Variables to Store Numbers and the Result of Their Multiplication

          #include <iostream>
          using namespace std;

          int main()
          {
              cout << "This program multiplies two numbers" << endl;

              cout << "Enter the first number: ";
              int firstNumber = 0;
              cin >> firstNumber;

              cout << "Enter the second number: ";
              int secondNumber = 0;
              cin >> secondNumber;

              // Multiply two numbers, store result in a variable
              int multiplicationResult = firstNumber * secondNumber;

              // Display result
              cout << firstNumber << " x " << secondNumber;
              cout << " = " << multiplicationResult << endl;

              return 0;
          }

          If you wanted to, you could condense the declaration of these three variables to one line of code that looks like this:

          #include <iostream>
          using namespace std;

          int main()
          {
              int firstNumber = 0, secondNumber = 0, multiplicationResult = 0;

              return 0;
          }

          Understanding the Scope of a Variable

          In C++, the scope of a variable refers to the region of the program where the variable is recognized and can be used. The scope determines where a variable is accessible and where it is not.

          Listing 3.2 Demonstrating the Scope of the Variables

          #include <iostream>
          using namespace std;

          void MultiplyNumbers()
          {
              cout << "Enter the first number: ";
              int firstNumber = 0;
              cin >> firstNumber;

              cout << "Enter the second number: ";
              int secondNumber = 0;
              cin >> secondNumber;

              // Multiply two numbers, store result in a variable
              int multiplicationResult = firstNumber * secondNumber;

              // Display result
              cout << firstNumber << " x " << secondNumber;
              cout << " = " << multiplicationResult << endl;
          }
          int main()
          {
              cout << "This program multiplies two numbers" << endl;

              // Call the function that does all the work
              MultiplyNumbers();

              // cout << firstNumber << " x " << secondNumber;
              // cout << " = " << multiplicationResult << endl;

              return 0;
          }

          Global Variables

          In C++, global variables are declared outside any function or class, making them accessible throughout the entire program. They have a global scope, allowing them to be accessed by any function in the same file or across multiple files when declared as extern. However, excessive use of global variables can lead to code maintenance challenges and is generally discouraged in favor of encapsulation and modular design.

          Listing 3.3 Using Global Variables

          #include <iostream>
          using namespace std;

          // Declare three global integers
          int firstNumber = 0;
          int secondNumber = 0;
          int multiplicationResult = 0;

          void MultiplyNumbers()
          {
              cout << "Enter the first number: ";
              cin >> firstNumber;

              cout << "Enter the second number: ";
              cin >> secondNumber;

              // Multiply two numbers, store result in a variable
              multiplicationResult = firstNumber * secondNumber;

              // Display multiplicationResult
              cout << "Displaying from MultiplyNumbers(): ";
              cout << firstNumber << " x " << secondNumber;
              cout << " = " << multiplicationResult << endl;
          }
          int main()
          {
              cout << "This program multiplies two numbers" << endl;

              // Call the function that does all the work
              MultiplyNumbers();

              cout << "Displaying from main(): ";

              // This line will now compile and work!
              cout << firstNumber << " x " << secondNumber;
              cout << " = " << multiplicationResult << endl;

              return 0;
          }

          Avoiding Overflow Errors by Selecting Correct Data Types in C++

          Overflow errors occur in C++ when a mathematical operation results in a value that's too big or too small for the chosen data type. This can lead to unpredictable behavior, crashes, and inaccurate results. Selecting the correct data type can help prevent such errors.

          Here's how choosing the right data type helps avoid overflow:

          1. Data types have size limitations: Each data type (e.g., int, float) has a specific range of values it can hold. Exceeding this range during an operation leads to overflow.
          2. Overflow behavior is undefined: C++ doesn't guarantee how an overflow will manifest. It might wrap around to the minimum value, yield unexpected results, or even crash the program.
          3. Matching data type to expected range: Choosing a data type that can comfortably hold the expected range of values in your program reduces the risk of overflow.

          Listing 3.4 Examples of the Ill Effects of Signed and Unsigned Integer Overflow Errors

          #include <iostream>
          using namespace std;

          int main()
          {
              unsigned short uShortValue = 65535;
              cout << "unsigned short 65535 + 1 = ";
              cout << ++uShortValue << endl;

              short signedShort = 32767;
              cout << "signed short 32767 + 1 = ";
              cout << ++signedShort << endl;

              return 0;
          }

        • Determining the Size of a Variable by Using sizeof()

          Size is the amount of memory that the compiler reserves when a programmer declares avariable to hold the data assigned to it. The size of a variable depends on its type, and C++ has a very convenient operator called sizeof() that tells you the size, in bytes, of a variable or a type.

          Listing 3.5 Finding the Sizes of Standard C++ Variable Types

          #include <iostream>

          int main()
          {
              using namespace std;
              cout << "Computing the size of inbuilt variable types" << endl;

              cout << "sizeof bool: " << sizeof(bool) << endl;
              cout << "sizeof char: " << sizeof(char) << endl;
              cout << "sizeof unsigned short int: " << sizeof(unsigned short) << endl;
              cout << "sizeof short int: " << sizeof(short) << endl;
              cout << "sizeof unsigned long int: " << sizeof(unsigned long) << endl;
              cout << "sizeof long: " << sizeof(long) << endl;
              cout << "sizeof int: " << sizeof(int) << endl;
              cout << "sizeof uns. long long: " << sizeof(unsigned long long) << endl;
              cout << "sizeof long long: " << sizeof(long long) << endl;
              cout << "sizeof unsigned int: " << sizeof(unsigned int) << endl;
              cout << "sizeof float: " << sizeof(float) << endl;
              cout << "sizeof double: " << sizeof(double) << endl;

              cout << "The output changes with compiler, hardware and OS" << endl;

              return 0;
          }

          Avoid Narrowing Conversion Errors by Using List Initialization

        • Automatic Type Inference Using auto

          auto is a magic keyword in C++ that lets the compiler automatically deduce the type of a variable based on its initialization. This can make your code cleaner and more readable, as you don't need to explicitly write the type every time.

          #include <iostream>
          using namespace std;

          int main()
          {
              int x = 10;         // Explicit type declaration
              auto y = 10;        // Compiler infers type as int based on initializer
              auto sum = 2 + 3;   // Compiler infers type as int (result of expression)
              auto pi = 3.14;     // Compiler infers type as double (literal value)
              auto str = "Hello"; // Compiler infers type as const char* (string literal)
              auto &ref = str;    // Compiler infers type as const char& (reference to const char*)
              return 0;
          }

          Listing 3.6 Using the auto Keyword and Relying on the Compiler’s Type-Inference Capabilities

          #include <iostream>
          using namespace std;

          int main()
          {
              auto coinFlippedHeads = true;
              auto largeNumber = 2500000000000;

              cout << "coinFlippedHeads = " << coinFlippedHeads << ", ";
              cout << "sizeof(coinFlippedHeads) = " << sizeof(coinFlippedHeads);
              cout << endl
                   << "largeNumber = " << largeNumber << ", ";
              cout << "sizeof(largeNumber) = " << sizeof(largeNumber) << endl;

              return 0;
          }

        • Using typedef to Substitute a Variable’s Type

          In C++, typedef allows you to create an alias for an existing data type, essentially substituting the type with a more descriptive or convenient name. This can be helpful for several reasons:

          1. Readability: Imagine using a long, complex type like unsigned long long repeatedly in your code. Using a typedef like ULongLong makes the code easier to read and understand.
          2. Code Clarity: typedef can clarify the meaning of a variable by using a descriptive name that reflects its purpose. For example, typedef int Age immediately tells you what the Age variable represents.
          3. Self-Documenting Code: Well-chosen typedef names can serve as documentation within the code, explaining the purpose and constraints of specific variables.

          Here are some examples of using typedef to substitute a variable's type:

          #include <iostream>
          using namespace std;
          typedef unsigned long long ULongLong; // Substitute ULongLong for unsigned long long
          typedef int Temperature;              // Now "Temperature" instead of generic "int"
          typedef struct
          {
              char name[32];
              int age;
          } Person; // Define a new type called "Person"

          int main()
          {
              ULongLong maxPopulation = 10000000;  // Now code becomes more readable
              Temperature boilingPointWater = 100; // Code clearly shows the variable's purpose
              Person john = {"John Doe", 30};      // Create a "Person" variable named john
              return 0;
          }

          Things to remember:

          1. typedef doesn't change the underlying data type, just its name.
          2. Use typedef sparingly to avoid cluttering your code.
          3. Choose clear and concise names for your typedefs.
